# Progressive Income Tax Calculator

A small TypeScript project created to practice programming logic, input validation, progressive tax calculations, data modeling, automated testing, and frontend development.

The current version uses a simplified fictional tax system. It is an educational calculator and must not be used for real financial or tax decisions.

## Project goals

The project started as a simple exercise in progressive tax logic:

- receive an income;
- validate the input;
- divide the income across tax brackets;
- calculate the tax produced by each bracket;
- calculate total tax, net income, and effective tax rate;
- display the result in the terminal and in a web interface;
- verify the calculation with automated tests.

The long-term goal is to evolve the project through multiple stages:

1. Generic progressive tax calculator
2. Complete browser interface
3. Configurable tax brackets
4. Better result and bracket breakdown
5. Support for different tax systems
6. Possible Brazilian income tax calculation mode

A future Brazilian version would require official and updated tax rules, deductions, exemptions, calculation bases, and clear documentation of the supported tax year.

## Current tax rules

The current educational brackets are:

| Income portion | Tax rate |
|---|---:|
| From 0 to 2,000 | 0% |
| From 2,000 to 3,000 | 10% |
| From 3,000 to 5,000 | 20% |
| Above 5,000 | 30% |

The calculator is progressive. Each rate applies only to the portion of income inside its corresponding bracket.

For example, an income of `6,000` is divided as follows:

| Bracket | Taxable portion | Rate | Tax |
|---|---:|---:|---:|
| 0вЂ“2,000 | 2,000 | 0% | 0 |
| 2,000вЂ“3,000 | 1,000 | 10% | 100 |
| 3,000вЂ“5,000 | 2,000 | 20% | 400 |
| Above 5,000 | 1,000 | 30% | 300 |

Total tax: `800`

## File responsibilities

### `src/index.ts`

Terminal entry point.

It is responsible for:

- reading the income from the terminal;
- converting the input from text to a number;
- validating the income;
- calling the tax calculation;
- displaying the result in the terminal.

### `src/web.ts`

Browser entry point.

It is responsible for:

- finding the HTML elements;
- listening for user interactions;
- reading the income typed by the user;
- converting the input into a number;
- validating the input;
- calling the tax calculation;
- displaying the result on the webpage;
- displaying and clearing validation errors.

### `src/taxCalculator.ts`

Contains the main application logic.

It is responsible for:

- validating income values;
- applying the progressive tax brackets;
- calculating the total tax;
- calculating the net income;
- calculating the effective tax rate;
- calculating the tax generated by each bracket;
- returning the complete calculation result.

### `src/taxRules.ts`

Contains the fixed tax configuration.

It stores:

- bracket minimum values;
- bracket maximum values;
- tax rates.

This file should contain tax-rule data, not terminal or browser behavior.

### `src/types.ts`

Contains the TypeScript types used by the project.

Examples include:

- the structure of a tax bracket;
- the structure of a complete tax calculation result;
- the type of the bracket tax list.

### `tests/taxCalculator.test.ts`

Contains automated tests for:

- valid income values;
- invalid income values;
- bracket boundaries;
- progressive tax behavior;
- total tax;
- net income;
- effective tax rate;
- tax generated by each bracket;
- large income values;
- decimal income values.

## Requirements

To run the project, the machine needs:

- Git
- Node.js
- npm

npm is normally installed together with Node.js.

No global TypeScript, Vite, Vitest, or `tsx` installation is required. The project dependencies are installed locally through `npm install`.

## Running the project on a new machine

### 1. Install Git

Check whether Git is installed:

```bash
git --version
```

If the command is not recognized, install Git from the official Git website.

After installing it, close and reopen the terminal.

### 2. Install Node.js and npm

Check whether Node.js and npm are installed:

```bash
node --version
npm --version
```

If either command is not recognized, install the current Node.js LTS version.

After installation, close and reopen the terminal, then run the version commands again.

### 3. Clone the repository

```bash
git clone <REPOSITORY_URL>
```

Replace `<REPOSITORY_URL>` with the repository URL.

Example:

```bash
git clone https://github.com/USERNAME/REPOSITORY.git
```

### 4. Enter the project directory

```bash
cd "Mini-Project Progressive Income Tax Calculator"
```

Quotation marks are useful because the folder name contains spaces.

### 5. Install the dependencies

```bash
npm install
```

This command reads `package.json` and `package-lock.json`, then downloads the required dependencies into the local `node_modules` folder.

Do not copy `node_modules` from another machine. Run `npm install` instead.

### 6. Check the TypeScript code

```bash
npm run typecheck
```

When no TypeScript errors are found, the command normally finishes without additional output.

### 7. Run the automated tests

```bash
npm test
```

This runs the Vitest test suite.

### 8. Run the terminal version

```bash
npm run dev
```

The program will request an income in the terminal.

Example:

```text
Type your income: 6000
```

### 9. Run the web version

```bash
npm run web
```

Vite will start a local development server and show an address similar to:

```text
http://localhost:5173
```

Open the displayed address in a browser.

To stop the development server, return to the terminal and press:

```text
Ctrl + C
```

## Troubleshooting

### `node` or `npm` is not recognized

Node.js is not installed, or the terminal has not been restarted after installation.

Install Node.js, restart the terminal, and check:

```bash
node --version
npm --version
```

### `git` is not recognized

Install Git, restart the terminal, and check:

```bash
git --version
```

### Dependencies are missing

Run:

```bash
npm install
```

### The website does not open

Start Vite:

```bash
npm run web
```

Use the exact local address displayed in the terminal.

### A port is already in use

Vite may automatically choose another port. Open the exact address displayed in the terminal instead of assuming port `5173`.

### TypeScript displays import errors

Run:

```bash
npm install
npm run typecheck
```

Also verify that local imports follow the module configuration used by the project.

### Tests fail

Run:

```bash
npm test
```

Read the expected and received values reported by Vitest.

A failed test may mean:

- the implementation contains a bug;
- a tax rule changed;
- the expected value in the test is incorrect;
- the function return structure changed without updating the tests.

### The browser page loads but the button does nothing

Check the browser developer console.

Common causes include:

- an incorrect HTML element ID;
- an incorrect `querySelector`;
- a missing `<script type="module" src="/src/web.ts"></script>` line;
- a Node-only import inside `web.ts`;
- an uncaught JavaScript error.

### Results disappear after invalid input

Do not erase the `textContent` of a container that contains the result elements.

Prefer hiding the result section or clearing only the individual output fields.

## Brazilian tax mode considerations

A real Brazilian income tax calculator is not just a replacement of the bracket values.

It may need to consider:

- the supported tax year;
- gross income versus taxable calculation base;
- exemptions;
- legal deductions;
- dependents;
- social security deductions;
- simplified deductions;
- monthly versus annual calculations;
- additional reductions and special rules;
- official updates from Receita Federal.

A Brazilian version should clearly state:

- which year it supports;
- whether it calculates monthly or annual tax;
- which deductions are supported;
- which cases are not supported;
- which official sources were used;
- that the result is educational and not professional tax advice.
